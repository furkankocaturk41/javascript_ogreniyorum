GIT Versiyon Kontrol Sistemi Nedir?
Versiyon Kontrol Sistemi veya Kaynak Kod Yönetim Sistemi 
Proje için tüm dökümanları izler,projenin snapShot özelliği ile o anki halini alabiliriz(Commit)
En çok kullanılan komutlar;
git init
git add
git commit
git status
git push
git pull
git clone
git checkout
git rm
----------------------------------------------------
git init; Henüz versiyon kontrolü altında olmayan bir projenin dizininde, boş bir git deposu oluşturmak için kullanılır. projemize eklerken 1 kere yazmak yeterli
git add;  Yeni eklenen veya üzerinde değişiklik yapılan dosyaları staged ortamına göndermek için kullanılır.
          Tek seferde bütün dosyaları eklemek için ise:
          git add . veya git add * veya git add -A .       Buradaki -A (all) tümü anlamındadır. . ise tüm dosya uzantılarını ifade eder.
git rm;   git rm --cached index.html
git status; Üzerinde çalışılan projenin o anki durumu hakkında bilgi verir. Yapılan değişiklikler, eklenen ve silinen dosyalar gibi bilgiler listelenir.
git commit; staged ortamına alınan dosyaların Local Repository’e gönderilmesidir. En iyi uygulama yöntemi her kayıt sırasında yapılan değişiklikleri açıklayıcı bir mesaj eklemektir. 
            git commit -m "ilk commit mesajı"    Buradaki -m (message) mesaj anlamındadır.
git log;    Projedeki commit geçmişini görüntülememizi sağlar. Bütün commit'ler, id'si, yazarı, tarihi ve mesajı ile beraber listelenir.
git branch; Local veya remote repository üzerinde yeni bir branch (dal) eklemek, silmek veya listelemek için kullanılır.
            branch eklemek için; git branch (branch_name) 
            *Tüm uzak ve yerel branch'lari listelemek için; git branch -a
            branch silmek için; git branch -d (branch_name)
git checkout; Branch’ler arası veya commit'ler arası geçiş yapmak istediğimizde kullanılır.
git push;  Projemizde aldığımız commit'leri, remote repository'e gönderir.
           git push origin master
           Burada bahsi geçen origin remote repository’nin kök dizinini belirtir ve sabit bir isimdir. master ise sizin çalıştığınız branch (dal)’ı belirtir.
           remote mastery; $ git remote add origin http://uzak_deponun_adresi.git
git clone; Mevcut bir Remote Repository'de bulunan dosyaların bilgisayarımızda bir kopyasının oluşturulmasını sağlar.
           git clone (remote_URL)
git pull;
git diff;
